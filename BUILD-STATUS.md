# ZametkiGraph — статус сборки MVP

**Дата:** 31.05.2026. **Архитектор:** Тимур. **Руки:** Уолтер.

## Что сделано (все 7 функций MVP из ТЗ)
1. ✅ Создать / открыть / удалить заметку (`.md`)
2. ✅ Редактировать markdown-текст
3. ✅ Связи `[[заметка]]` — парсинг + кликабельный переход (панель «Связи в заметке»; если цели нет — создаётся)
4. ✅ Обратные связи (backlinks) — панель снизу
5. ✅ Боковая панель — список заметок + превью
6. ✅ Поиск по заметкам (заголовок + текст)
7. ✅ Хранение в локальной папке (по умолчанию `Documents/ZametkiGraphVault`, можно выбрать свою через «Выбрать папку»)

## Сборка
- **Mac:** `** BUILD SUCCEEDED **`, приложение запускается, создаёт хранилище с демо-заметками.
- **iPhone:** код компилируется (typecheck против iOS Simulator SDK 18.5 — чисто). НО на этом Mac **не установлен ни один iOS-симулятор runtime** → запустить телефонную версию пока негде. Лечится: `xcodebuild -downloadPlatform iOS` (качает ~7 ГБ) либо через Xcode → Settings → Components.

## Как собрать заново (без терминальной возни)
```
cd /Users/serv/Desktop/walter-cc/timur/notes-app
xcodegen generate
open ZametkiGraph.xcodeproj   # дальше ⌘R в Xcode
```
Или Mac-сборка из CLI:
```
xcodebuild -project ZametkiGraph.xcodeproj -scheme ZametkiGraph-macOS -destination 'platform=macOS' build
```

## Архитектура (принцип ТЗ соблюдён)
- Данные = нейтральные `.md` файлы (читаются в любом редакторе, в т.ч. в самом Obsidian).
- Хранение (`VaultStore`) отделено от синхры — слой синхры в MVP отсутствует, ядро под него не завязано.
- Общий код Mac+iOS (`Sources/Shared`), платформенные различия только в bookmark-доступе к папке (`#if os(macOS)`).
